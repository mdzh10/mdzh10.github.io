Designing a Social Media Platform for Mentally Challenged People

Muhammad Zubair Hasan
Course Project - System Analysis, Design & Development (SADD) Feb 2020 - July 2020
Slides Report

An Accessible Social Platform specifically designed for mentally challenged individuals, featuring simplified interactions, enhanced safety measures, and comprehensive support systems. A SADD course initiative focused on inclusive design and user empowerment.

1. Introduction

This project, titled 'Designing a Social Website for Mentally Challenged People,' aims to create an accessible, safe, and engaging digital platform for individuals with psychological challenges. The focus is on building trust, simplifying interactions, and fostering social inclusion through tailored content and community support. The initiative was driven by Group 4 (B) from the System Analysis and Design course.

2. Objectives

  • Analyzing existing social platforms and support systems for mentally challenged individuals
  • Gathering data through research papers, field interviews, and direct observations
  • Designing a mobile-first, accessible social platform
  • Developing an intuitive, user-friendly interface with core functionalities such as news feed, emergency contacts, chat, and profiles

Key Features

1. Emergency Help System

  • 24/7 Emergency Contact: Immediate access to emergency services
  • One-Click Help: Simplified emergency button for quick assistance
  • Location Sharing: Automatic location sharing with emergency contacts
  • Caregiver Alerts: Instant notification to designated caregivers

2. Restricted Internet Access

  • Content Filtering: Safe browsing with inappropriate content blocking
  • Time Management: Controlled internet usage with time limits
  • Whitelist System: Pre-approved safe websites and content
  • Parental Controls: Caregiver oversight of online activities

3. Communication Features

  • Video Calls: Face-to-face communication with family and friends
  • Message Sending: Text-based messaging with simplified interface
  • File Sharing: Safe sharing of photos and documents
  • Location Sharing: Share location with trusted contacts

4. Expert Consultation

  • Discussion Sessions: Group sessions with mental health professionals
  • Individual Consultations: One-on-one sessions with psychiatrists/psychologists
  • Resource Library: Educational materials and coping strategies
  • Progress Tracking: Monitor mental health progress over time

5. Accessibility Features

  • Simple Navigation: Intuitive and easy-to-use interface
  • Large Text Options: Adjustable font sizes for better readability
  • Color Contrast: High contrast themes for visual accessibility
  • Voice Commands: Voice-activated features for hands-free operation

System Design & Analysis

1. Context Diagrams

  • System Boundaries: Clear definition of system scope and interactions
  • External Entities: Identification of users, caregivers, and service providers
  • Data Flows: Mapping of information exchange between system components
  • Process Modeling: Detailed workflow analysis for each feature

2. User Interface Design

  • Wireframes: Low-fidelity UI sketches for layout planning
  • Prototypes: Interactive mockups for user testing
  • User Experience: Focus on simplicity and ease of use
  • Accessibility Guidelines: WCAG compliance for inclusive design

3. Workflow Analysis

  • User Journeys: Step-by-step user interaction flows
  • Error Handling: Graceful error recovery and user guidance
  • Safety Protocols: Emergency response procedures
  • Data Security: Privacy protection and secure communication

Technical Implementation

1. Frontend Development

  • Responsive Design: Mobile-first approach for accessibility
  • Progressive Web App: Offline functionality and app-like experience
  • Accessibility Framework: ARIA labels and keyboard navigation
  • Custom UI Components: Specialized components for target users

2. Backend Architecture

  • User Management: Secure authentication and authorization
  • Content Filtering: AI-powered inappropriate content detection
  • Real-time Communication: WebSocket-based messaging and video calls
  • Emergency System: High-priority emergency response handling

3. Security & Privacy

  • Data Encryption: End-to-end encryption for sensitive communications
  • Privacy Controls: Granular privacy settings for user data
  • Audit Logging: Comprehensive activity tracking for safety
  • Compliance: HIPAA and other relevant privacy regulations

Course Integration

  • System Design: Architecture and component design
  • Development Methodology: Agile development with iterative design
  • Testing & Validation: User acceptance testing with target audience
  • Documentation: Complete system documentation and user guides

Impact & Results

  • Safety Enhancement: Comprehensive emergency response system
  • Social Inclusion: Enabling meaningful social connections
  • Professional Support: Direct access to mental health professionals
  • Educational Value: Comprehensive system design documentation
  • Real-world Application: Practical solution for a critical need
